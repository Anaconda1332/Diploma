# Пошаговая реализация ML-модели для оценки риска мошенничества

Кратко: мы обучаем небольшую нейросеть на **признаках устройства** (root, контакты, Wi‑Fi, биометрия и т.д.), сохраняем её в формате **TensorFlow Lite** и запускаем **прямо на телефоне** в твоём SDK. Ниже — шаги с нуля, без предположения, что ты уже знаком с ML.

---

## Что нужно установить (один раз)

### 1. Python 3.8+

- Скачай и установи с [python.org](https://www.python.org/downloads/).
- При установке отметь **"Add Python to PATH"**.
- В терминале проверь: `python --version` или `py -3 --version`.

### 2. Виртуальное окружение (рекомендуется)

В папке проекта открой терминал и выполни:

```bash
cd ml
python -m venv venv
```

Затем активируй окружение:

- **Windows (cmd):** `venv\Scripts\activate.bat`
- **Windows (PowerShell):** `venv\Scripts\Activate.ps1`
- **macOS/Linux:** `source venv/bin/activate`

В начале строки должно появиться `(venv)`.

### 3. Установка библиотек

В том же терминале (с активированным `venv`):

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

Это установит TensorFlow, pandas, numpy, scikit-learn — всё, что нужно для обучения и экспорта модели.

**Если появилась ошибка про компилятор (Visual Studio, gcc, "Preparing metadata did not run successfully"):** pip пытается собрать пакет из исходников, а на Windows для этого нужен компилятор C. Проще запретить сборку и ставить только готовые бинарники:

```bash
pip install --upgrade pip
pip install --only-binary :all: -r requirements.txt
```

Если и так не ставится (например, для Python 3.13 ещё нет готовых wheel'ов) — установи **Python 3.11** или **3.12** с [python.org](https://www.python.org/downloads/), заново создай виртуальное окружение в папке `ml` и повтори установку.

## Шаг 1: Понять, какие признаки мы используем

Модель получает на вход **один вектор чисел** — по одному числу на каждый признак. Порядок признаков **фиксированный** (одинаковый в Python и в Android).

| № | Признак | Описание | Откуда в SDK |
|---|--------|----------|----------------|
| 0 | `is_rooted` | Рут: 0 или 1 | `isDeviceRooted()` |
| 1 | `log_contact_count` | ln(1 + кол-во контактов) | `getContactCount()` |
| 2 | `wifi_networks_count` | Кол-во Wi‑Fi сетей (макс. 50) | `getWiFiNetworks().size` |
| 3 | `has_biometric` | Есть ли биометрия: 0 или 1 | `getBiometricInfo()["can_authenticate"]` |
| 4 | `sdk_version_norm` | Версия Android / 34 | `Build.VERSION.SDK_INT` |
| 5 | `is_call_active` | Идёт ли звонок: 0 или 1 | `isCallActive()` |
| 6 | `hour_of_day_norm` | Час дня / 23 (0…1) | Передаётся из приложения или 0 |
| 7 | `day_of_week_norm` | День недели / 6 (0…1) | Передаётся из приложения или 0 |

Итого **8 признаков**. На выходе модель выдаёт одно число от 0 до 1 — **вероятность мошенничества** (risk score).

---

## Шаг 2: Сгенерировать учебные данные

У нас нет реальной разметки «мошенник / не мошенник», поэтому для демо и защиты проекта мы **генерируем синтетические данные** по простым правилам:

- **Мошенник (label=1):** часто рут, мало контактов, много Wi‑Fi сетей, нестабильные комбинации.
- **Обычный пользователь (label=0):** реже рут, больше контактов, более «нормальные» значения.

Запусти:

```bash
python generate_synthetic_data.py
```

В папке `ml` появятся файлы:

- `train_data.csv` — таблица с признаками и столбцом `label` (0 или 1).

Открой `train_data.csv` в Excel или блокноте и посмотри: столбцы — это признаки, последний столбец — метка. Так модель будет «понимать», какие комбинации мы считаем рискованными.

## Шаг 3: Обучить модель и экспортировать в TFLite

Скрипт делает по шагам:

1. Читает `train_data.csv`.
2. Нормализует признаки (вычитает среднее, делит на стандартное отклонение) — так нейросеть обучается стабильнее.
3. Обучает небольшую нейросеть (MLP): два скрытых слоя, на выходе одно число (вероятность).
4. Сохраняет обученную модель в формате **TensorFlow Lite** (`.tflite`) и параметры нормализации в **JSON** (чтобы на телефоне применять ту же нормализацию).

Запусти:

```bash
python train_and_export.py
```

В папке `ml` появятся:

- `fraud_risk_model.tflite` — модель для Android.
- `scaler_params.json` — средние и масштабы по каждому признаку (нужны для нормализации на устройстве).
- В консоли будет выведен **AUC** (метрика качества: чем ближе к 1, тем лучше).

## Шаг 4: Скопировать модель и параметры в Android-проект

1. Открой в проекте папку:
   ```
   fraud-detection-sdk/src/main/assets/
   ```
2. Скопируй в неё:
   - `ml/fraud_risk_model.tflite`
   - `ml/scaler_params.json`

Если папки `assets` нет — создай её вручную: `fraud-detection-sdk/src/main/assets/`.

## Шаг 5: Использовать модель в приложении

В коде приложения:

1. Получить экземпляр SDK и скорера:
   ```kotlin
   val sdk = FraudDetectionSDK.getInstance(context)
   val riskScorer = FraudRiskScorer.getInstance(context)
   ```
2. Собрать признаки (можно передать час и день недели из приложения или оставить `null`):
   ```kotlin
   val score = riskScorer.evaluateRisk(sdk, hourOfDay = null, dayOfWeek = null)
   ```
3. Интерпретировать `score` (0.0…1.0):
   - **0.0–0.3** — низкий риск
   - **0.3–0.7** — средний (например, запросить подтверждение)
   - **0.7–1.0** — высокий риск

Подробнее см. класс `FraudRiskScorer` в SDK и пример в демо-приложении.

## Шаг 6: Проверка в Android Studio

1. Убедись, что в `fraud-detection-sdk/src/main/assets/` лежат `fraud_risk_model.tflite` и `scaler_params.json`.
2. Собери и запусти приложение на эмуляторе или устройстве (Run ▶).
3. Выдай разрешения приложению (Check Permissions → разрешить контакты, телефон, локацию, Wi‑Fi при запросе).
4. Нажми **«Рассчитать риск»** в карточке «ML Risk Score (TensorFlow Lite)».
5. Должны появиться:
   - в карточке: **Score: 0.xxx** и **Уровень риска: Низкий / Средний / Высокий**;
   - Toast с тем же значением.

Если видишь **«Модель не загружена»** — проверь, что файлы скопированы в `assets/` и что папка называется именно `assets` (без опечаток). После добавления файлов в `assets/` пересобери проект (Build → Rebuild Project).

## Кратко: что за что отвечает

- **generate_synthetic_data.py** — создаёт учебный CSV с признаками и метками (для демо без реальных данных).
- **train_and_export.py** — обучает нейросеть, считает AUC, сохраняет `.tflite` и `scaler_params.json`.
- **FraudRiskScorer** (Kotlin) — на устройстве загружает модель и параметры, собирает вектор из SDK, нормализует его, прогоняет через модель и возвращает risk score.

Если что-то пойдёт не так на каком-то шаге — напиши, на каком именно (например: «Шаг 3, ошибка при pip install» или «Шаг 5, падает при вызове evaluateRisk»), и разберём по шагам.
